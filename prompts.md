# Codex 요청 문장 예시

이 문서는 Codex에게 작업을 요청할 때 바로 쓸 수 있는 문장 모음입니다.

프로젝트 상황에 맞게 일부 단어만 바꿔서 사용하면 됩니다.

## 프로젝트 시작

```txt
프로젝트 문서 먼저 확인하고, 현재 목표와 부족한 정보를 요약해줘.
아직 파일은 수정하지 말고 확인 필요한 질문부터 정리해줘.
```

```txt
AGENTS.md, context/, brand/, workflow/ 문서를 먼저 읽고 작업 방향을 짧게 제안해줘.
그다음 바로 진행해도 되는 작업과 확인이 필요한 작업을 나눠줘.
```

## 문서 작성

```txt
context/project-brief.md와 context/business-context.md 기준으로 제안서 초안을 작성해줘.
확정되지 않은 내용은 확인 필요라고 표시해줘.
```

```txt
이 문서를 비개발자 대표가 이해할 수 있게 쉽게 다시 써줘.
긴 문장은 줄이고, 어려운 용어는 첫 등장 때 설명해줘.
```

## PPT 제작

```txt
context와 brand 문서를 먼저 확인한 뒤 PPT 초안을 만들어줘.
한 슬라이드에는 하나의 핵심 메시지만 담고, 완료 후 검수 이미지와 요약을 만들어줘.
```

```txt
이 제안서를 10분 발표용 PPT 구조로 바꿔줘.
슬라이드 제목만 읽어도 흐름이 이해되게 구성해줘.
```

## 코드 작업

```txt
먼저 관련 파일을 확인하고, 변경 범위와 검증 방법을 짧게 설명한 뒤 수정해줘.
기존 구조와 스타일을 우선 따라줘.
```

```txt
이 변경에 필요한 테스트나 실행 확인 방법을 먼저 정하고 작업해줘.
테스트가 과하면 더 가벼운 검증 방법을 제안해줘.
```

## 검수 요청

```txt
workflow/review-checklist.md 기준으로 결과물을 검수해줘.
문제가 있으면 우선순위순으로 정리하고, 바로 수정 가능한 것은 수정해줘.
```

```txt
이 결과물을 대표가 봤을 때 이해하기 어려운 부분 중심으로 검토해줘.
과장된 표현, 근거 없는 숫자, 확인이 필요한 내용을 찾아줘.
```

## 수정 요청

```txt
방금 결과물에서 말투만 더 쉽게 바꿔줘.
내용 구조는 유지하고, 어려운 용어와 긴 문장만 줄여줘.
```

```txt
디자인 방향은 유지하되, 정보가 더 잘 보이도록 레이아웃을 정리해줘.
중요한 문구는 이미지가 아니라 편집 가능한 텍스트로 유지해줘.
```

## 개인 명령어

```txt
/grill-me
이 기획에서 가장 위험한 가정 하나만 짚고, 해결 방향도 짧게 제안해줘.
```

```txt
/to-prd
지금까지 논의한 내용을 PRD로 정리해줘.
확정된 내용과 확인 필요한 내용을 구분해줘.
```

```txt
/rubber-duck
이 코드의 흐름을 줄 단위로 설명하고, 잠재적인 버그를 짚어줘.
아직 코드는 수정하지 말아줘.
```
