# 노트북 발표 시연 가이드

## 시연 목표

중고거래 경매 웹사이트 안에서 시세 그래프가 보조 기능으로 쓰이는 흐름을 보여준다.

발표자는 물품 등록, 입찰, 낙찰, 관리자 검수 순서로 화면을 이동한다.

## 실행 방법

가상환경이 준비되어 있다면 아래 순서로 실행한다.

```powershell
.\.venv\Scripts\python.exe scripts\reset_demo.py
.\.venv\Scripts\python.exe -m flask --app app run --host 127.0.0.1 --port 5000
```

브라우저에서 아래 주소로 접속한다.

```text
http://127.0.0.1:5000
```

## 시연 순서

1. 메인 화면
   - 경매 웹사이트가 주 기능임을 보여준다.
   - 시세 그래프는 가격 판단을 돕는 보조 기능이라고 설명한다.

2. 물품 등록 화면
   - 대표 상품을 선택한다.
   - 저장 데이터 기반 시세 그래프를 확인한다.
   - 시작가와 입찰 단위를 입력해 경매를 등록한다.

3. 경매 상세 화면
   - 현재 입찰가와 시세 그래프를 함께 보여준다.
   - 입찰자 이름과 입찰가를 입력한다.
   - 입찰 내역이 추가되는지 확인한다.

4. 낙찰 결과 화면
   - 경매 마감 후 낙찰 결과를 확인한다.
   - 결제 기능은 과제 범위에서 제외했다고 설명한다.

5. 관리자 검수 화면
   - 판매자가 관리자에게 물건을 보낸다.
   - 관리자가 검수 상태를 바꾼다.
   - 검수 후 구매자에게 전달하는 흐름을 보여준다.

## 발표 시 주의할 점

- 웹사이트는 실제 사이트를 실시간 크롤링하지 않는다.
- 미리 저장한 CSV와 SQLite 데이터로 시세를 보여준다.
- 회원 기능과 결제 기능은 제외 범위다.
- 실제 배송 연동도 제외하고 상태 화면으로만 표현한다.
- 그래프 라이브러리는 로컬 정적 파일로 포함되어 있어 인터넷 연결이 없어도 화면이 뜬다.

## 확인 필요

- 발표 노트북에서 Python과 가상환경이 정상 실행되는지 확인 필요
- 발표 전 `data/app.db`가 샘플 데이터로 초기화되어 있는지 확인 필요
- 사이트별 크롤링 정책은 제출 전 최신 기준으로 재확인 필요
