# 프로젝트 브리프

## 작업 목표

중고거래 사이트의 단점과 기존 경매 사이트의 불편함을 보완한 경매 기반 중고거래 웹사이트를 제작합니다.

이 사이트는 판매자와 구매자가 직접 거래하는 구조가 아니라, 사이트 업체가 중간에서 상품 검수와 금액 정산을 담당하는 안전 거래 구조를 목표로 합니다.

## 대상 독자

- 개발 팀원
- 프로젝트 발표를 준비하는 팀원
- Codex 또는 AI 도구로 구현 작업을 진행하는 사용자
- 프로젝트를 검토하는 교육 과정 강사 또는 평가자

## 핵심 메시지

이 프로젝트는 단순 중고거래 사이트가 아니라, 안전 거래와 시세 기반 가격 판단을 결합한 경매 플랫폼입니다.

사용자는 가격 흥정 없이 입찰 또는 즉시구매를 통해 거래하고, 사이트 업체는 상품 검수와 정산을 통해 사기와 분쟁을 줄입니다.

## 발표 또는 사용 상황

이 프로젝트는 AI 데이터 분석 과정 팀 프로젝트로 사용합니다.

발표에서는 다음 흐름을 중심으로 설명합니다.

1. 기존 중고거래 사이트의 문제
2. 기존 경매 사이트의 불편함
3. 우리 서비스의 차별화
4. 경매 거래 플로우
5. 주요 화면 구성
6. 기대 효과
7. 향후 확장 가능성

## 반드시 포함할 내용

### 1. 기존 서비스 문제

중고거래 사이트의 주요 단점은 다음과 같습니다.

- 사기 위험
- 물건 파손 문제
- 판매자와 구매자 간 잦은 가격 흥정
- 시세 확인의 어려움
- 거래 취소 시 보상 기준 부족

기존 경매 사이트의 주요 불편함은 다음과 같습니다.

- 사용 방식이 복잡함
- 시세 정보가 부족함
- 입찰 기준이 직관적이지 않음
- 중고거래 사용자에게 친숙하지 않은 화면 구성

### 2. 서비스 차별화 방향

우리 서비스는 이베이, 코베이, 크림의 장단점을 참고하여 다음 방향으로 차별화합니다.

- 경매 방식으로 가격 흥정을 줄입니다.
- 사이트 업체가 중간에서 상품을 검수합니다.
- 구매자는 사이트 업체에 금액을 송금합니다.
- 판매자는 사이트 업체를 통해 정산받습니다.
- 상품 등록 시 시세 정보를 제공합니다.
- 거래 취소 원인에 따라 경고와 포인트 보상을 적용합니다.

### 3. 경매 시스템 플로우

#### 상품 등록

1. 판매자가 웹사이트에 경매품을 등록합니다.
2. 판매자는 제품 사진, 제품명, 브랜드명을 입력합니다.
3. 사이트는 제품명과 브랜드명을 기준으로 제품 시세 정보를 제공합니다.
4. 판매자는 제공된 정보를 확인합니다.
5. 판매자는 선호 가격을 입력합니다.
   - 즉시구매가
   - 경매시작가
6. 판매자는 경매 마감 시간을 선택합니다.
   - 1일
   - 3일
   - 7일
   - 15일

#### 입찰

1. 구매자는 웹사이트에 올라온 제품을 확인합니다.
2. 구매자는 상품 상세 화면에서 현재 입찰가, 즉시구매가, 남은 시간을 확인합니다.
3. 구매자는 즉시구매 또는 입찰을 선택합니다.
4. 입찰은 현재 입찰가에 비율 또는 최소 증가액을 더하는 방식으로 진행합니다.
5. 경매 시간이 종료되거나 즉시구매가 발생하면 가격이 확정됩니다.
6. 가격이 확정되면 판매자와 구매자에게 알림을 보냅니다.

#### 거래 진행

1. 판매자는 가격 확정 후 1~3일 안에 사이트 업체로 상품을 배송합니다.
2. 구매자는 가격 확정 후 1~3일 안에 사이트 업체로 금액을 송금합니다.
3. 사이트 업체는 상품 상태를 확인합니다.
4. 상품 상태에 문제가 없으면 사이트 업체가 구매자에게 상품을 배송합니다.
5. 사이트 업체는 수수료를 제외한 금액을 판매자에게 송금합니다.

#### 거래 취소

판매자 원인으로 거래가 취소되는 경우:

- 판매자가 1~3일 안에 상품을 보내지 않으면 경매를 취소합니다.
- 판매자에게 경고 1회를 부여합니다.
- 판매자가 경고 3회를 받으면 제재합니다.
- 구매자에게 1,000포인트를 제공합니다.
- 포인트는 배송비 할인에 사용할 수 있습니다.

구매자 원인으로 거래가 취소되는 경우:

- 구매자가 1~3일 안에 금액을 송금하지 않으면 경매를 취소합니다.
- 구매자에게 경고 1회를 부여합니다.
- 구매자가 경고 3회를 받으면 제재합니다.
- 판매자에게 1,000포인트를 제공합니다.
- 포인트는 배송비 할인에 사용할 수 있습니다.

### 4. 시세 정보

제품 시세는 다음 서비스에서 수집한 정보를 바탕으로 제공합니다.

- 중고나라
- 번개장터
- 당근마켓

시세 정보는 제품명과 브랜드명을 기준으로 조회합니다.

표시 정보:

- 최저가
- 평균가
- 최고가
- 현재 경매시작가
- 즉시구매가

시세는 1차원 게이지 형식으로 제공합니다.

시세 데이터가 부족한 경우에는 `시세 정보 부족`으로 표시합니다.

## 성공 기준

### 사용자 관점

- 판매자는 시세를 참고하여 상품 가격을 설정할 수 있습니다.
- 구매자는 흥정 없이 입찰 또는 즉시구매를 할 수 있습니다.
- 판매자와 구매자는 직접 금전 거래를 하지 않아도 됩니다.
- 거래 취소 시 책임과 보상 기준이 명확합니다.

### 서비스 관점

- 중고거래의 사기 위험을 줄이는 구조가 드러납니다.
- 경매 방식의 재미와 긴장감이 보입니다.
- 메인 화면에서 중요한 상품을 빠르게 확인할 수 있습니다.
- 시세 기반 가격 판단이 가능합니다.

### 개발 관점

- HTML 파일은 `templates/` 폴더에 정리되어 있습니다.
- 프로젝트 요구사항이 `context/project-brief.md`에 정리되어 있습니다.
- Codex 작업 기준은 `AGENTS.md`에서 확인할 수 있습니다.
- GitHub 저장소와 브랜치 기준이 명확합니다.
