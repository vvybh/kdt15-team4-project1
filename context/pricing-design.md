# 시세 그래프와 가격 계산 설계

## 목적

이 문서는 시세 그래프와 가격 참고 정보의 기준을 정리한다.
시세 그래프는 경매 웹사이트 안에서 판매자와 구매자의 가격 판단을 돕는 기능이다.
판매자는 시작가를 정할 때 사용한다.
구매자는 현재 입찰가가 적절한지 판단할 때 사용한다.

웹사이트는 가격을 확정해서 말하지 않는다.
대신 수집한 중고거래 데이터를 기준으로 참고 시세와 가격 범위를 보여준다.
계산은 실시간 크롤링 결과가 아니라 저장된 SQLite 데이터를 기준으로 한다.
CSV는 샘플 작성과 SQLite 적재 준비에 사용한다.

## 기본 원칙

- 같은 상품끼리 먼저 비교한다.
- 같은 상품 데이터가 부족하면 비슷한 조건의 상품까지 넓혀 비교한다.
- 평균 가격보다 중간 가격을 우선 사용한다.
- 너무 낮거나 높은 이상치는 계산에서 제외한다.
- 데이터가 부족하면 `데이터 부족`이라고 표시한다.
- 최신 데이터가 없으면 `최근 수집 데이터 기준`이라고 표시한다.
- 계산 기준은 화면과 발표에서 설명할 수 있어야 한다.

중간 가격은 가격을 낮은 순서로 정렬했을 때 가운데에 있는 값이다.
평균보다 이상치 영향을 덜 받는다.

## 사용 데이터

가격 계산은 `price_snapshots` 테이블을 기준으로 한다.
웹사이트는 이 테이블에 저장된 데이터만 읽는다.
크롤러가 실패해도 기존 데이터를 유지한다.
초기 샘플 데이터 형식은 `context/sample-data-format.md`를 따른다.

사용 필드는 아래와 같다.

| 필드명 | 용도 |
| --- | --- |
| product_key | 같은 상품을 묶는 기준 |
| price | 가격 계산 |
| observed_at | 그래프 x축과 최신 데이터 판단 |
| source | 사이트별 출처 표시 |
| trade_status | 판매중, 판매완료 구분 |
| item_condition | 상품 상태 비교 |

`market_items`의 브랜드, 상품명, 모델명, 구성품, 지역 정보는 비교 기준을 만들 때 함께 사용한다.
`product_key`는 CSV 샘플 데이터와 SQLite 데이터에서 같은 값으로 유지한다.

## 동일 상품 기준

동일 상품은 아래 조건을 우선으로 판단한다.

1. 카테고리가 같다.
2. 브랜드가 같거나 비어 있지 않게 일치한다.
3. 상품명 핵심 단어가 같다.
4. 모델명이 있으면 모델명이 같다.

모델명이 있는 상품은 모델명을 가장 강한 기준으로 본다.
모델명이 없는 상품은 브랜드와 상품명 핵심 단어를 기준으로 묶는다.

예시는 아래와 같다.

| 입력 상품 | 동일 상품으로 볼 수 있는 예 |
| --- | --- |
| 나이키 후드티 | 나이키 후드, Nike 후드티 |
| 아이폰 13 128GB | iPhone 13 128GB |
| 크리넥스 휴지 24롤 | 크리넥스 24롤 휴지 |

## product_key 생성 기준

`product_key`는 같은 상품을 묶기 위한 내부 키다.

생성 순서는 아래와 같다.

1. 카테고리를 정리한다.
2. 브랜드를 정리한다.
3. 상품명에서 핵심 단어를 뽑는다.
4. 모델명이 있으면 모델명을 붙인다.
5. 공백과 특수문자를 줄인다.

예시는 아래와 같다.

| 카테고리 | 브랜드 | 상품명 | 모델명 | product_key 예시 |
| --- | --- | --- | --- | --- |
| 의류 | Nike | 나이키 후드티 | 없음 | 의류:nike:후드티 |
| 전자기기 | Apple | 아이폰 13 128GB | A2633 | 전자기기:apple:iphone13:128gb |
| 생필품 | 크리넥스 | 크리넥스 휴지 24롤 | 없음 | 생필품:크리넥스:휴지:24롤 |

## 비슷한 조건 기준

동일 상품 안에서도 상태와 구성에 따라 가격이 달라진다.
그래서 아래 조건을 함께 비교한다.

### 상품 상태

상품 상태는 아래 단계로 맞춘다.

| 상태 그룹 | 설명 |
| --- | --- |
| 새상품 | 사용하지 않은 상품 |
| 거의 새것 | 사용 횟수가 적고 상태가 좋은 상품 |
| 사용감 적음 | 정상 사용 가능하고 큰 흠이 없는 상품 |
| 사용감 있음 | 사용 흔적이 뚜렷한 상품 |
| 하자 있음 | 고장, 오염, 파손 등이 있는 상품 |
| 상태 확인 필요 | 상태를 알 수 없는 상품 |

비교할 때는 같은 상태 그룹을 먼저 사용한다.
데이터가 부족하면 바로 옆 상태 그룹까지 포함한다.
예를 들어 `사용감 적음`은 `거의 새것`, `사용감 있음`과 함께 볼 수 있다.

### 구성품

구성품은 아래처럼 나눈다.

| 구성품 그룹 | 설명 |
| --- | --- |
| 풀세트 | 박스, 설명서, 부속품이 모두 있는 경우 |
| 일부 누락 | 구성품 일부가 없는 경우 |
| 단품 | 본품만 있는 경우 |
| 구성 확인 필요 | 구성품 정보가 없는 경우 |

풀세트와 단품은 가격 차이가 날 수 있다.
구성품 정보가 없으면 계산에는 포함하되, 화면에는 `구성 확인 필요`로 표시한다.

### 거래 상태

가격 계산에는 `판매완료` 데이터를 우선 사용한다.
판매완료 데이터가 부족하면 `판매중` 데이터를 보조로 사용한다.

판매완료는 실제 거래 가격에 더 가깝다.
판매중은 희망 가격일 수 있으므로 참고용으로 본다.

### 기간

최근 데이터를 우선 사용한다.
기본 기간은 최근 6개월로 둔다.
6개월 안에 데이터가 부족하면 최근 12개월까지 넓힌다.

## 데이터 선택 순서

가격 계산에 사용할 데이터는 아래 순서로 찾는다.

1. 같은 product_key, 같은 상태 그룹, 최근 6개월 데이터
2. 같은 product_key, 비슷한 상태 그룹, 최근 6개월 데이터
3. 같은 product_key, 모든 상태 그룹, 최근 12개월 데이터
4. 같은 카테고리와 핵심 단어가 비슷한 상품, 최근 12개월 데이터

각 단계에서 데이터가 충분하면 다음 단계로 넓히지 않는다.
데이터가 너무 적으면 화면에 `데이터 부족`을 표시한다.

## 최소 데이터 기준

가격 계산에는 최소 5개 이상의 가격 데이터가 있는 것을 권장한다.
5개보다 적으면 가격을 단정하지 않는다.

표시 방식은 아래와 같다.

| 데이터 개수 | 표시 방식 |
| --- | --- |
| 0~2개 | 데이터 부족 |
| 3~4개 | 참고용 가격만 표시 |
| 5개 이상 | 시세 범위와 참고 가격 표시 |

이 숫자는 과제 구현을 위한 기준이다.
실제 서비스 기준은 더 많은 데이터가 필요할 수 있다.

## 이상치 제거 기준

이상치는 전체 가격 흐름과 너무 다른 값이다.
아래 데이터는 계산에서 제외하거나 확인 대상으로 분리한다.

- 가격이 0원인 데이터
- `무료나눔`, `가격제안`처럼 숫자 가격이 없는 데이터
- 같은 상품인데 가격이 지나치게 낮거나 높은 데이터
- 부품만 판매하는 글
- 고장품이나 하자품인데 상태가 표시되지 않은 글
- 광고성 글

계산 방식은 아래처럼 한다.

1. 가격을 낮은 순서로 정렬한다.
2. 낮은 쪽 25% 지점과 높은 쪽 25% 지점을 구한다.
3. 그 범위에서 너무 벗어난 값을 이상치 후보로 본다.
4. 이상치 후보는 시세 계산에서 제외한다.

낮은 쪽 25% 지점과 높은 쪽 25% 지점은 데이터를 4등분했을 때의 경계값이다.
발표에서는 `일반적인 가격 범위`라고 설명해도 된다.

## 가격 계산 방식

### 참고 시세

참고 시세는 이상치를 제거한 뒤 중간 가격으로 계산한다.

```txt
참고 시세 = 비슷한 조건 상품들의 중간 가격
```

### 시세 범위

시세 범위는 낮은 쪽 25% 지점부터 높은 쪽 25% 지점까지로 본다.

```txt
시세 범위 = 낮은 쪽 25% 가격 ~ 높은 쪽 25% 가격
```

### 판매 시작가 참고

판매 등록 화면에서는 시작가 참고값을 보여준다.
경매는 입찰을 유도해야 하므로 참고 시세보다 약간 낮은 값이 자연스럽다.

```txt
추천 시작가 = 시세 범위의 낮은 쪽 가격
```

단, 판매자가 직접 시작가를 수정할 수 있어야 한다.

### 구매 입찰 판단

경매 상세 화면에서는 현재 입찰가를 시세 범위와 비교한다.

| 현재 입찰가 | 표시 문구 |
| --- | --- |
| 시세 범위보다 낮음 | 시세보다 낮은 편 |
| 시세 범위 안 | 일반적인 시세 범위 |
| 시세 범위보다 높음 | 시세보다 높은 편 |

이 문구는 구매자의 판단을 돕는 안내다.
구매를 권유하거나 가격을 보장하는 표현은 쓰지 않는다.

## 시세 그래프 설계

시세 그래프는 날짜별 가격 흐름을 보여준다.

### 그래프 기본 구성

- x축: 날짜
- y축: 가격
- 점: 수집된 개별 가격 데이터
- 선: 날짜별 중간 가격 흐름
- 색상: 사이트별 출처 구분
- 기준선: 참고 시세
- 범위 표시: 시세 범위

### 판매 등록 화면 그래프

판매 등록 화면에서는 아래 정보를 보여준다.

- 최근 가격 분포
- 참고 시세
- 추천 시작가
- 데이터 개수
- 데이터 출처

판매자는 이 정보를 보고 시작가를 입력한다.

### 경매 상세 화면 그래프

경매 상세 화면에서는 아래 정보를 보여준다.

- 최근 가격 분포
- 참고 시세
- 현재 입찰가
- 현재 입찰가가 시세 범위 안인지 여부
- 데이터 출처

구매자는 이 정보를 보고 입찰 여부를 판단한다.

## 화면 표시 문구

화면에는 아래처럼 표시한다.

```txt
참고 시세: 35,000원
일반 시세 범위: 30,000원 ~ 42,000원
추천 시작가: 30,000원
데이터 기준: 최근 6개월, 12건
수집 기준: 최근 수집 데이터 기준
출처: 당근마켓, 번개장터, 중고나라
```

데이터가 부족하면 아래처럼 표시한다.

```txt
데이터가 부족해 참고 시세를 계산하기 어렵습니다.
비슷한 상품의 가격만 참고용으로 표시합니다.
```

## 구현 함수 기준

`services/price_service.py`에서 아래 기능을 담당한다.

| 함수 | 역할 |
| --- | --- |
| build_product_key(item) | 상품을 묶는 키 생성 |
| normalize_condition(value) | 상품 상태 정리 |
| find_comparable_items(item) | 같은 상품과 비슷한 조건 데이터 찾기 |
| remove_outliers(prices) | 이상치 제거 |
| calculate_price_summary(items) | 참고 시세와 시세 범위 계산 |
| build_chart_data(items) | Chart.js에서 쓸 데이터 생성 |
| compare_bid_to_market(current_bid, summary) | 현재 입찰가 판단 |

## 확인 필요 항목

### 1. 데이터 부족 기준

1. 권장안: 5개 이상이면 계산, 3~4개면 참고용 표시
   - 과제 구현과 발표에 적당하다.
   - 데이터 부족 상황도 자연스럽게 처리할 수 있다.
2. 간단안: 3개 이상이면 바로 계산
   - 구현이 쉽다.
   - 시세 신뢰도가 낮아질 수 있다.
3. 확장안: 카테고리별로 최소 데이터 개수를 다르게 둔다.
   - 더 정교하다.
   - 설명과 구현이 복잡해진다.

### 2. 기간 기준

1. 권장안: 최근 6개월을 기본으로 하고 부족하면 12개월까지 확장
   - 최신성과 데이터 수를 균형 있게 볼 수 있다.
2. 간단안: 기간 제한 없이 전체 데이터를 사용
   - 구현이 쉽다.
   - 오래된 가격이 섞일 수 있다.
3. 확장안: 최근 데이터에 더 높은 비중을 준다.
   - 최신 시세에 더 가깝다.
   - 계산 설명이 어려워진다.

### 3. 상품 상태 비교 범위

1. 권장안: 같은 상태를 먼저 보고, 부족하면 옆 상태까지 포함
   - 중고 가격 차이를 반영하면서 데이터 부족도 줄일 수 있다.
2. 간단안: 상태를 구분하지 않는다.
   - 구현이 쉽다.
   - 가격 기준이 흐려질 수 있다.
3. 확장안: 상태별 보정값을 둔다.
   - 더 정교하다.
   - 보정값 근거를 설명해야 한다.
