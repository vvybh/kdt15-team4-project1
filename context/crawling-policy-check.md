# 크롤링 정책 확인

확인일: 2026-05-29

이 문서는 팀 프로젝트 범위에서 크롤링 가능성을 점검하기 위한 자료다. 법률 자문은 아니다. 실제 제출 전에는 팀원이 각 사이트의 최신 약관과 robots.txt를 다시 확인해야 한다.

## 결론

웹사이트 시연은 실제 사이트를 실시간으로 크롤링하지 않는다.

수집해 둔 CSV와 SQLite 데이터를 사용한다. 실제 크롤러는 과제 설명용 또는 제한된 수집용으로만 둔다.

이유는 단순하다. 기술적으로 페이지를 가져올 수 있어도, 사이트 정책과 개인정보 문제가 따로 있기 때문이다.

## 공통 수집 원칙

- 웹사이트는 저장된 데이터만 읽는다.
- 실시간 크롤링은 발표 시연 흐름에 넣지 않는다.
- 개인정보는 수집하지 않는다.
- 연락처, 채팅 내용, 사용자 계정 정보, 위치 상세 정보는 저장하지 않는다.
- robots.txt에서 제한한 경로는 접근하지 않는다.
- 로그인, 우회 접속, 자동 반복 요청은 하지 않는다.
- 수집 실패 시 기존 샘플 데이터를 사용한다.
- 사이트 정책이 바뀌면 `확인 필요`로 표시한다.

## 크롤러 코드에 반영할 규칙

1. 수집 대상 URL은 허용 경로인지 먼저 확인한다.
2. 요청 간격을 둔다.
3. 가져온 HTML에서 필요한 필드만 추출한다.
4. 개인정보로 보이는 값은 저장하지 않는다.
5. 원본 HTML 전체 저장은 기본값으로 하지 않는다.
6. 실패하면 예외를 기록하고 다음 항목으로 넘어간다.

## 확인 필요

1. 각 사이트의 최신 이용약관 재확인
   - 권장 선택지: 발표 제출 전날 한 번 더 확인
   - 간단 선택지: 현재 문서 기준으로 제출
   - 확장 선택지: 사이트별 고객센터에 문의

2. 실제 크롤링 수행 여부
   - 권장 선택지: 과제 시연은 샘플 데이터만 사용
   - 간단 선택지: 크롤러 코드는 실행하지 않고 구조만 설명
   - 확장 선택지: 허용 경로에서 소량만 직접 수집

3. 개인정보 제거 기준
   - 권장 선택지: 연락처, 계정명, 상세 위치는 전부 저장 제외
   - 간단 선택지: 가격과 제목만 저장
   - 확장 선택지: 개인정보 탐지 규칙을 코드로 추가
