# 작업 지침

## 목표

이 프로젝트의 목표는 사용자가 요청한 산출물을 명확하고 실용적인 형태로 완성하는 것이다.

대상 독자와 사용 상황을 먼저 이해하고, 그에 맞는 말투와 구조를 사용한다.

## 말투

- 쉬운 우리말을 쓴다.
- 긴 문장보다 짧은 문장을 쓴다.
- 어려운 용어는 첫 등장 때 한 문장으로 풀어 설명한다.
- 확정되지 않은 내용은 `확인 필요`라고 표시한다.
- 근거 없는 숫자나 성과를 만들지 않는다.

## 작업 원칙

- 문제를 먼저 설명하고, 해결 방법을 나중에 설명한다.
- 기능보다 결과를 먼저 말한다.
- 사람이 판단할 일과 도구가 도와줄 일을 구분한다.
- 기존 파일 구조와 작성 방식을 먼저 확인한다.
- 관련 없는 리팩터링이나 변경은 하지 않는다.

## 작업 전 확인할 파일

작업을 시작하기 전에 가능한 경우 아래 파일을 먼저 확인한다.

1. `context/project-brief.md`
2. `context/business-context.md`
3. `brand/DESIGN.md`
4. `workflow/review-checklist.md`

## 완료 보고

작업이 끝나면 아래 내용을 짧게 보고한다.

- 만든 결과물
- 수정한 파일
- 검증한 내용
- 확인이 필요한 내용
