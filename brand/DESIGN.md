# 디자인 방향

## 전체 분위기

- 신뢰감
- 실무 중심
- 과장 없는 전문성
- 중고거래 사용자가 빠르게 이해할 수 있는 구성
- 가격 정보와 경매 상태가 잘 보이는 화면

## 디자인 원칙

- 기능보다 사용자가 얻는 결과를 먼저 보여준다.
- 경매 거래 흐름을 가장 먼저 보이게 배치한다.
- 시세 그래프는 판매와 구매 판단이 필요한 화면 안에 배치한다.
- 가격, 그래프, 입찰 상태가 한 흐름으로 이해되게 구성한다.
- 한 화면에는 하나의 핵심 행동을 중심으로 둔다.
- 발표자료는 문제, 해결, 구현 결과가 자연스럽게 이어지게 만든다.
- 확인되지 않은 수치나 성과는 넣지 않는다.

## 색상

- 배경: 흰색 또는 밝은 회색 계열 권장
- 본문: 어두운 회색 또는 검은색 계열 권장
- 강조: 입찰 버튼과 핵심 가격에 사용할 색상 확인 필요
- 상승 가격: 빨간색 계열 권장
- 하락 가격: 파란색 계열 권장
- 경고 또는 문제 표시: 주황색 또는 빨간색 계열 권장

최종 색상 코드는 확인 필요.

## 글꼴

- 제목: 굵고 읽기 쉬운 한글 글꼴 권장
- 본문: 긴 설명을 읽기 쉬운 한글 글꼴 권장
- 웹사이트 글꼴: 확인 필요
- 발표자료 글꼴: 확인 필요

## 웹사이트 화면 구성

- 메인 화면: 검색창, 인기 물품, 진행 중인 경매
- 등록 화면: 물품명, 설명, 사진, 시작가, 경매 마감 시간, 시세 그래프
- 경매 상세 화면: 물품 정보, 현재 입찰가, 남은 시간, 입찰 버튼, 시세 그래프
- 검색 또는 물품 상세 화면: 물품명, 가격 그래프, 평균 가격, 적정 가격 참고 정보
- 관리자 검수 화면: 낙찰 물품, 수령 상태, 검수 결과, 구매자 발송 상태
- 데이터 확인 화면: 확인 필요

## 웹 구현 기준

- 중심 기술은 Python과 HTML이다.
- 웹 프레임워크는 Flask를 사용한다.
- 데이터 저장은 SQLite를 사용한다.
- 그래프는 Chart.js를 사용한다.
- 화면은 HTML에서 읽기 쉽게 구성한다.
- 경매 상태를 먼저 보여준다.
- 그래프는 사용자가 가격을 판단해야 하는 위치에 함께 보여준다.

## 발표자료 레이아웃

- 한 슬라이드에는 하나의 핵심 메시지를 담는다.
- 제목만 읽어도 흐름이 이해되게 한다.
- 표, 도형, 텍스트는 가능하면 편집 가능한 객체로 유지한다.
- 중요한 문구는 이미지 안에 넣지 않는다.
- 그래프는 실제 수집 데이터 또는 예시 데이터 여부를 표시한다.
- PPT는 10~15장 안에서 구성한다.
- 10분 발표 기준으로 슬라이드당 설명 시간을 짧게 잡는다.

## 발표자료 권장 구성

1. 프로젝트 제목과 한 줄 소개
2. 중고거래 가격 판단 문제
3. 경매 웹사이트 중심의 해결 방향
4. 데이터 수집 대상
5. 데이터 정리 방식
6. 경매 거래 흐름
7. 판매 등록 화면의 시세 그래프
8. 구매 입찰 화면의 시세 그래프
9. 관리자 검수 중개 흐름
10. 웹사이트 주요 화면
11. 팀원별 역할
12. 한계와 개선 방향
13. 마무리

## 이미지와 그래프 사용

- 이미지는 장식보다 이해를 돕는 용도로 사용한다.
- 특정 회사, 특정 제품, 실제 고객사를 떠올리게 하는 이미지는 피한다.
- 이미지 안에 긴 문장이나 중요한 설명을 넣지 않는다.
- 그래프에는 기준 기간, 데이터 출처, 물품명을 함께 표시한다.
- 예시 데이터라면 `예시 데이터`라고 표시한다.

## 확인 필요

- 서비스 이름
- 로고 사용 여부
- 대표 색상
- 사용할 글꼴
- 실제 웹사이트 화면 크기 기준
- 발표자료 템플릿 여부

## 웹사이트 화면 기준

- 노트북 발표 시연을 기본 기준으로 한다.
- 13~15인치 화면에서 메인 흐름이 한눈에 보여야 한다.
- 메인 화면은 모바일 앱처럼 좁게 보이지 않게 대시보드형 웹사이트로 구성한다.
- 경매 목록은 카드 나열보다 목록형 정보 구조를 우선한다.
- 모바일 화면에서도 무너지지 않도록 반응형을 유지한다.
